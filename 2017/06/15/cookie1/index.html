cookie(一)：图解cookie

发表于 2017-06-15 | 分类于 Javascript

话不多说，上图

图一：js获取cookie

关键词：document.cookie

  • js通过document.cookie这个代码来读取cookie
  • 由n个key=value;形式的键值对组成的一个字符串
  • document.cookie并不能获取cookie的 domain，path，expires 或 secure 标记

图二：cookie的属性选项

关键词：cookie的属性选项

  • cookie 的属性选项 expires、domain、path、secure、HttpOnly

来，排好队，一个一个介绍

先来段code：

document.cookie="user_name=EmmieHuang; expires=Thu, 25 Feb 201604:18:00 GMT; domain=liepini.com; path=/; secure; HttpOnly“;

expires

  • 含义：cookie失效日期（如此例指该user_name会在Thu, 25 Feb 2016 04:18:00之后失效，即会被浏览器自动删除）
  • 格式：GMT 格式（可以通过 new Date().toGMTString()或者 new Date().toUTCString() 来获得）
  • 不设置：也称session cookie，即有效期为session，浏览器关闭后就没有了

据此，可将cookie分为会话cookie(Session cookie)和持久性cookie(Persistent cookie)

domain 和 path：

  • 含义：domain和path一起来限制 cookie 能被哪些 URL 访问
  • 不设置： domain的默认值为设置该cookie的网页所在的域名，path默认值为设置该cookie的网页所在的目录

secure

  • 含义：指定后，cookie只有在使用SSL链接的时候才发送到服务器。如，cookie信息只能发送到https://www.liepin.com，而http://www.liepin.com的请求则不能发送cookie。

  • 不设置：默认为空，即不管是HTTPS协议还是HTTP协议的请求，cookie 都会被发送至服务端（domain和path都符合条件的情况下）

HttpOnly

这个不太常用啊，但是还是要了解一下的，看到了吗

  • 含义：当cookie带httpOnly选项时，客户端则无法通过js代码去访问（包括读取、修改、删除等）这个cookie

  • 不设置：默认是不设置这个值的，

在客户端是不能通过js代码去设置一个httpOnly类型的cookie的，这种类型的cookie只能通过服务端来设置

图三：js设置cookie

这是一个ajax请求的 Request Headers，符合条件的cookie会以这种方式发送到服务端。

关键词：js设置cookie

  • 创建：设置多个cookie的方法就是重复执行document.cookie = “key=name”；

    正确

    document.cookie = "name=Jonh";
document.cookie = "age=12";
document.cookie = "class=111";

    错误

    document.cookie = "name=Jonh; age=12; class=111";

    错误的方式会导致除了第一个，后面的都不会设置成功

  • 修改：重新赋值，旧的值会被新的值覆盖。但要注意一点，在设置新cookie时，path/domain这几个选项一定要旧cookie 保持一样。否则不会修改旧值，而是添加了一个新的 cookie。
  • 删除：也是重新赋值，只要将这个新cookie的expires 选项设置为一个过去的时间点就行了。但同样要注意，path/domain/这几个选项一定要旧cookie保持一样。

js并没有直接提供修改和删除cookie的方法，所以其原理都是重新赋值
